Scenario

3-person startup sprint

A 2-week sprint. Three developers. Three AI tools. One shared STATE_VECTOR.json in the repo.

AliceBackend - Claude Code
Builds API endpoints for user authentication and billing. Establishes database schema decisions, error handling patterns, and rate limiting strategy. Syncs after each session.
contextkeeper sync --agent claude
BobFrontend - GPT-4
Bootstraps Alice's state before starting. GPT-4 immediately knows the API contracts, error shapes, and auth flow. Builds React components that target the exact endpoints Alice created. No guessing, no mismatched types.
contextkeeper bootstrap --agent gpt-4
CarolDevOps - Cursor
Reads both Alice's and Bob's state vectors. Sets up CI/CD knowing the exact test structure, deployment requirements, and environment variables. Configures GitHub Actions that enforce the invariants Alice defined.
contextkeeper bootstrap --agent cursor

STATE_VECTOR.json becomes the team brain

Committed to the shared repo, the state vector captures every architectural decision, every rejected alternative, and every invariant - from every developer, across every AI tool. Pull the repo, bootstrap, and you have full team context in 60 seconds.

Governance

The actor model for teams

DEL-v2's Operator/Executor/Auditor roles map directly to how teams already work. The lead developer is the Operator. Individual developers are Executors. Code review is the Auditor role.

Role Team mapping What they can do What they cannot do
Operator Tech lead / PM Define tasks, set invariants, approve state transitions, perform manual validation Execute implementation directly
Executor Individual developer + their AI Write code, run tests, produce artifacts, report gate results Self-approve work, skip gates, seal milestones
Auditor Code reviewer / QA Review execution evidence, verify SHA integrity, validate invariants Execute code, modify state
Scenario

Code review with full context

The reviewer's superpower

A reviewer opens a PR. Instead of reading the diff cold, they load the handoff bundle. They see everything that led to this change:

  • Every architectural decision that shaped this code
  • Every alternative that was considered and rejected, with rationale
  • Every invariant that must hold after this change
  • The full experiment history (for data/ML PRs)
  • SHA256 verification that nothing was tampered with
code review workflow
