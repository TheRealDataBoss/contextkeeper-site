About

Built from real pain

contextkeeper was born from 18 months of AI-assisted development on a production codebase.

The Origin Story

From handoff chaos to a formal protocol

In 2024, I started building 3dpie - a production 3D data visualization engine using React and Three.js. I used AI assistants heavily: Claude for architecture, GPT for specific implementations, Cursor for quick edits.

Within weeks, I hit the wall. Every new session started from zero. Every model switch meant re-explaining decisions. Critical architectural choices were lost between conversations. The AI would contradict decisions made three sessions ago because it had no memory of them.

I started writing handoff documents manually. Then I formalized them. Then I built a state vector. Then a decision log. Then an invariant registry. Then a state machine to govern transitions. Then SHA verification to prevent tampering.

18 months later, that handoff system became contextkeeper. It's been tested across 1,099 tests, 244 end-to-end tests, and 29 sealed milestones in production. It works because it was built to solve a problem I hit every single day.
